# 37\_開発構造の選択

CryptoStoneは次の構造を採用する。

* STONE Token
* CryptoStone Gem NFT
* 12個のMining Pool Contract
* Pool Factory

この構造は、特定チェーンに依存しないデジタル宝石プロトコルを実装するための設計であり、初期には EVM互換スマートコントラクトネットワークをLaunch Networkとして活用する。

## 選択理由

| 選択                    | 理由                             |
| --------------------- | ------------------------------ |
| STONE Token           | 1つのマイニング資源と流動性構造を提供する。         |
| CryptoStone Gem NFT   | 各宝石の固有性と属性を表現する。               |
| 12 Mining Pools       | ストーン別独立鉱山構造を実装する。              |
| Pool Factory          | 同一の検証済みテンプレートでプールをデプロイする。      |
| Unified NFT Contract  | 1つのコレクションアイデンティティと取引データを集中させる。 |
| Verifiable Randomness | 運営者操作のない属性生成を可能にする。            |
| Finalize Mechanism    | 核心規則の変更を防ぐ。                    |

1. 第一に、STONEとGem NFTの役割を分離する。STONEはマイニングパワーのためのトークンであり、Gem NFTは採掘結果物である。
2. 第二に、宝石の固有性を保護する。各Gem NFTはtokenIdと属性組み合わせを持ち、発行後に属性が変更されない。
3. 第三に、ストーン別独立性を維持する。各マイニングプールは独立した供給量、マイニング周期、半減期を持つため、現実の個別鉱山概念をデジタルで表現するのに適している。
4. 第四に、NFTコレクションは1つに維持される。これはCryptoStoneのブランドと市場データを1つに集中させ、取引と希少度ランキングを統合的に管理できるようにする。
5. 第五に、将来的な拡張性が高い。CryptoStoneはEVM互換Launch Networkで開始でき、長期的には独立したAppchainまたはCryptoStone Mainnetへ拡張できる。
6. 第六に、中央化依存性を減らす。採掘と発行はサーバーではなくコントラクトによって実行され、誰でも条件を検証できる。
