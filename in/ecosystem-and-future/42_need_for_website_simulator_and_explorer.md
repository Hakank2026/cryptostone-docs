# 42 Kebutuhan Website Simulator dan Explorer

Protokol CryptoStone sendiri beroperasi melalui smart contract dan data on-chain. Namun, antarmuka pengguna terpisah diperlukan agar pengguna dapat memahami dan berpartisipasi secara intuitif.

Karena itu, ekosistem CryptoStone dapat memerlukan alat berbasis web berikut.

| Tool | Role |
| ---- | ---- |
| Mining Simulator | Menghitung estimasi waktu mining berdasarkan jumlah STONX pengguna, pool yang dipilih, dan periode lock-up. |
| PoM Dashboard | Menampilkan nilai PoM pengguna berdasarkan pool, ambang PoM yang diperlukan, dan ketersediaan claim. |
| Pool Dashboard | Menampilkan jumlah yang telah ditambang, supply tersisa, Pool Difficulty, dan Scarcity Multiplier tiap stone pool. |
| Gem Explorer | Memungkinkan pengguna melihat attributes, rarity, tokenId, mining time, dan riwayat transaksi Gem NFT yang telah ditambang. |
| Rarity Explorer | Memungkinkan pengguna memeriksa rarity dan Probability Rarity Index berdasarkan kombinasi Weight, Color, Clarity, dan Cut. |
| Refinery Console | Memungkinkan pengguna memeriksa dan mengeksekusi kondisi Gem Refinement, tabel probabilitas, dan catatan refinement. |
| Protocol Status Page | Menampilkan informasi protokol penting seperti Open Market Allocation, No Admin Mint, total supply, tabel probabilitas, dan status pool dengan cara yang dapat diverifikasi. |

Alat ini tidak menggantikan fondasi kepercayaan inti protokol. Alat tersebut berfungsi sebagai antarmuka yang membantu pengguna memahami data on-chain dengan mudah. Dengan kata lain, esensi CryptoStone terletak pada contract dan aturan on-chain, sementara website dan explorer adalah alat untuk menafsirkan dan memvisualisasikannya.
