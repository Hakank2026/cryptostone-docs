# 40\_position\_of\_images\_and\_metadata

CryptoStone does not deny visual representation. Gemstone NFTs can be represented as images, 3D models, or visual card formats for user experience and collecting convenience.

However, images are not the essence of CryptoStone.

The essence of CryptoStone lies in the following data:

| Data                   | Description                         |
| ---------------------- | ----------------------------------- |
| stoneType              | Gemstone type                       |
| weight                 | Weight                              |
| colorGrade             | Color grade                         |
| clarityGrade           | Clarity grade                       |
| cutGrade               | Cut grade                           |
| rarityScore            | User-friendly rarity score          |
| probabilityRarityIndex | Probability-based rarity indicator  |
| minedAt                | Mining time                         |
| minedFromPool          | Mining pool from which it was mined |
| tokenId                | NFT unique identifier               |
| generation             | Original or refinement generation   |

Therefore, even if the image server is temporarily unavailable, the core attributes of a CryptoStone NFT should remain in an on-chain or verifiable data structure. This is designed to improve on the issue of existing NFTs relying excessively on external images or central servers.
