# 04\_协议信任原则

CryptoStone 追求一种不依赖特定创始人或运营方信任的结构。本协议的信任基础并非创始人的身份，而是公开代码、固定发行量、不可变更的概率表、No Admin Mint 结构、可验证随机性，以及任何人都可以确认的链上数据。

运营方不得能够向特定用户任意分配稀有 NFT，不得改变各宝石供应量，也不得操纵挖矿概率。CryptoStone 的目标是构建一种由协议规则，而非人的权威，来形成数字宝石稀缺性的结构。

| 信任原则                    | 说明                             |
| ----------------------- | ------------------------------ |
| Fixed Supply            | STONE 及各宝石 NFT 最大发行量预先固定。      |
| 100% Public Circulation | STONE 遵循 100% 公开流通结构。          |
| No Admin Mint           | 运营方不得任意发行 Gem NFT。             |
| Immutable Probability   | 宝石属性概率表必须公开，并在 finalize 后不得更改。 |
| Verifiable Randomness   | 宝石属性必须由可验证随机结构决定。              |
| On-chain Attributes     | Gem NFT 的核心属性必须记录为链上数据。        |
| Transparent Pools       | 各宝石池的挖矿量、剩余数量和难度必须公开可查。        |
| Open Verification       | 任何人都必须能够验证合约和挖矿规则。             |

这一结构表明，CryptoStone 并不是由特定运营方的裁量来运作，而是一个由公开规则和可验证代码驱动的去中心化数字宝石协议。
