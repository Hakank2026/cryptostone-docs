# 40\_图像与元数据的位置

CryptoStone 并不否定视觉表达。为了用户体验和收藏便利性，宝石 NFT 可以被表达为图像、3D 模型或视觉卡片形式。

但是，图像并不是 CryptoStone 的本质。

CryptoStone 的本质在于以下数据：

| 数据                     | 说明         |
| ---------------------- | ---------- |
| stoneType              | 宝石种类       |
| weight                 | 重量         |
| colorGrade             | 颜色等级       |
| clarityGrade           | 净度等级       |
| cutGrade               | 切工等级       |
| rarityScore            | 用户友好的稀有度分数 |
| probabilityRarityIndex | 基于概率的稀有度指标 |
| minedAt                | 挖矿时间       |
| minedFromPool          | 所来自的挖矿池    |
| tokenId                | NFT 唯一标识符  |
| generation             | 原始或精炼世代    |

因此，即使图像服务器暂时中断，CryptoStone NFT 的核心属性也应保留在链上或可验证数据结构中。这是为了补足现有 NFT 过度依赖外部图像或中央服务器的问题。
